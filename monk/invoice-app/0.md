# Base Solution Reference

## Schema (src/lib/types.ts)

```ts
type InvoiceStatus = "draft" | "open" | "paid" | "void" | "uncollectible";
type PaymentMethod = "credit_card" | "bank_transfer" | "check" | "cash";

interface LineItem {
  id: string;
  description: string;
  quantity: number;
  unit_price: number;       // in cents
  amount: number;           // quantity * unit_price, in cents
}

interface Invoice {
  id: string;
  invoice_number: string;
  status: InvoiceStatus;
  customer_id: string;
  customer_name: string;
  customer_email: string;
  line_items: LineItem[];
  subtotal: number;                       // in cents
  tax_rate: number;                       // e.g. 0.08 for 8%
  tax_amount: number;                     // in cents
  total: number;                          // in cents
  amount_due: number;                     // total - payments, in cents
  finalized_at: string | null;            // when draft -> open
  due_date: string | null;
  paid_at: string | null;
  voided_at: string | null;
  marked_uncollectible_at: string | null;
  memo?: string;
  created_at: string;
  updated_at: string;
}

interface Payment {
  id: string;
  invoice_id: string;
  amount: number;          // in cents
  method: PaymentMethod;
  paid_at: string;
  note?: string;
}

interface Customer {
  id: string;
  name: string;
  email: string;
  address?: string;
  created_at: string;
}

interface Database {
  invoices: Invoice[];
  customers: Customer[];
  payments: Payment[];
}
```

## API Endpoints

### GET /api/customers

Returns all customers.

```
Response: Customer[]
```

---

### GET /api/invoices

Returns all invoices.

```
Response: Invoice[]
```

---

### POST /api/invoices

Creates a new draft invoice. Server generates id, invoice_number, and calculates all totals.

```
Request body:
{
  customer_id: string,
  customer_name: string,
  customer_email: string,
  line_items: [{ description, quantity, unit_price }],
  tax_rate?: number,       // default 0
  memo?: string
}

Response (201): Invoice   // status = "draft", amount_due = total
```

---

### GET /api/invoices/:id

Returns a single invoice by id.

```
Response: Invoice
Errors:
  404 - "Invoice not found"
```

---

### PATCH /api/invoices/:id

Edit a draft invoice. Only drafts can be edited. Server recalculates subtotal, tax_amount, total, amount_due.

```
Request body (all optional):
{
  line_items?: LineItem[],
  memo?: string,
  tax_rate?: number,
  customer_id?: string,
  customer_name?: string,
  customer_email?: string
}

Response: Invoice (updated)
Errors:
  404 - "Invoice not found"
  400 - "Only draft invoices can be edited"
```

---

### DELETE /api/invoices/:id

Delete a draft invoice. Only drafts can be deleted.

```
Response: { success: true }
Errors:
  404 - "Invoice not found"
  400 - "Only draft invoices can be deleted"
```

---

### POST /api/invoices/:id/finalize

Transition draft -> open. Sets finalized_at, due_date (default Net 30).

```
Request body (optional):
{
  due_date?: string   // ISO date, defaults to 30 days from now
}

Response: Invoice (status = "open")
Errors:
  404 - "Invoice not found"
  400 - "Only draft invoices can be finalized"
```

---

### POST /api/invoices/:id/payments

Record a payment on an open invoice. If amount_due reaches 0, status flips to "paid".

```
Request body:
{
  amount: number,          // in cents, must be > 0 and <= amount_due
  method?: string,         // default "credit_card"
  note?: string
}

Response (201):
{
  payment: Payment,
  invoice: Invoice         // updated amount_due, possibly status = "paid"
}

Errors:
  404 - "Invoice not found"
  400 - "Can only record payments on open invoices"
  400 - "Payment must be between 1 and {amount_due}"
```

---

### GET /api/invoices/:id/payments

List all payments for an invoice.

```
Response: Payment[]
```

---

### POST /api/invoices/:id/void

Void an invoice. Allowed from: open, uncollectible, draft. Blocked from: paid, already void. Sets amount_due to 0.

```
Response: Invoice (status = "void", amount_due = 0)
Errors:
  404 - "Invoice not found"
  400 - "Invoice is already voided"
  400 - "Cannot void a paid invoice - issue a credit note instead"
```

---

### POST /api/invoices/:id/mark-uncollectible

Write off an open invoice. Only open invoices can be marked uncollectible. Keeps amount_due as-is.

```
Response: Invoice (status = "uncollectible")
Errors:
  404 - "Invoice not found"
  400 - "Only open invoices can be marked uncollectible"
```

---

## Valid Transitions

```
draft -> draft           PATCH  (edit)
draft -> open            POST   /finalize
draft -> (deleted)       DELETE
open  -> paid            POST   /payments (when amount_due hits 0)
open  -> void            POST   /void
open  -> uncollectible   POST   /mark-uncollectible
uncollectible -> void    POST   /void
paid  -> terminal
void  -> terminal
```

## Frontend (src/app/page.tsx)

The UI fetches `GET /api/invoices` on load and after every action. Each action handler follows the same pattern:

```ts
async function handleAction(id: string) {
  await fetch(`/api/invoices/${id}/action`, { method: "POST" });  // write
  fetchInvoices();  // re-read to refresh UI
}
```

Actions shown per status:
- **draft**: Finalize, Delete
- **open**: Record Payment, Void, Write Off
- **uncollectible**: Void
- **paid**: "Settled" (no actions)
- **void**: "Voided" (no actions)
